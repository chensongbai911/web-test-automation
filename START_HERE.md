## 🎯 立即开始的 3 个步骤

### 第 1 步：加载插件到 Chrome (1 分钟)

```
1. 打开Chrome浏览器
2. 访问: chrome://extensions/
3. 开启"开发者模式"（右上角开关）
4. 点击"加载已解压的扩展程序"
5. 选择: d:\test-auto\web-test-automation 文件夹
6. ✓ 完成！插件已加载
```

验证成功：

- Chrome 右上角出现插件图标
- 点击图标弹窗打开正常

### 第 2 步：打开插件并配置 (1 分钟)

```
1. 点击Chrome右上角的插件图标
2. 输入要测试的网址，例如:
   https://www.baidu.com
   或者
   https://www.jd.com
3. 确认配置选项已勾选：
   ☑ 测试交互按钮
   ☑ 监控API请求
   ☑ 捕获截图
   ☑ 捕获控制台日志
```

### 第 3 步：运行测试并查看报告 (2-5 分钟)

```
1. 点击"开始测试"按钮
2. 等待新标签页打开并加载网页
3. 观察弹窗中的实时进度
4. 测试完成后，点击"查看报告"
5. 在新标签页中查看详细报告
```

## 🐛 常见问题 30 秒快答

**Q: 插件无法加载？**
A: 检查 `chrome://extensions/` 是否成功加载，查看是否有错误信息。

**Q: 测试为什么失败？**
A: 查看报告中的"失败元素"，确认网站是否正常访问。

**Q: 如何自定义测试？**
A: 编辑 `src/content-script.js` 文件中的函数。

**Q: 数据存储在哪里？**
A: Chrome 的 Local Storage，关闭浏览器时不会删除。

**Q: 支持哪些浏览器？**
A: Chrome 及 Chromium 系列浏览器。

**Q: 会修改网站数据吗？**
A: 大多数情况只是读操作，建议在测试环境运行。

✓ 定期测试重要网站
